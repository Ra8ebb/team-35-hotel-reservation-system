🏨 Team 35 Hotel Reservation System

Team 35's Hotel Reservation System is a robust, enterprise-grade Java application designed to streamline hospitality management. It features comprehensive role-based access, real-time room availability tracking, and automated check-in/check-out workflows. Backed by a live SQL database and featuring automated periodic reporting, the system ensures a seamless, secure, and data-driven booking experience from start to finish.



✨ Key Features

Role-Based Access Control: Tailored functionalities and permissions for Guests, Receptionists, and Administrators.



Live SQL Database Integration: Real-time data persistence, ensuring all reservations, user profiles, and hotel states are securely stored and instantly accessible.



Automated CSV Export Service: A scheduled background service that periodically exports critical system data to CSV format for reporting, analytics, and secure backups.



Smart Reservation Management: End-to-end booking, check-in, and check-out logic seamlessly synced with live room availability to prevent double-booking.



Strict Data Validation: Built-in security and sanity checks to prevent invalid chronological dates, empty fields, and negative financial inputs.



Custom Exception Handling: Clean, predictable business logic execution utilizing tailored error handling.



🛠️ Technologies Used

Core Language: Java



Database: SQL (Relational Database) \& JDBC



Architecture: Object-Oriented Programming (OOP) Principles



Data I/O: Automated CSV Generation \& File Management



🚀 How to Run

Prerequisites

Java Development Kit (JDK) installed.



A running SQL database server (e.g., MySQL, PostgreSQL, etc.).



Your preferred Java IDE (IntelliJ IDEA, Eclipse, or VS Code).
